## Dev Notes

### Architecture Patterns and Constraints

**Testing Strategy:**
- **Unit Tests**: Test individual services, controllers, guards, interceptors in isolation
  - Mock all external dependencies (Prisma, I18nService, Logger, external APIs)
  - Focus on business logic and edge cases
  - Fast execution (no database, no network)
  - High coverage target: Services 80%+, Controllers 70%+

- **E2E Tests**: Test complete request-response flow through HTTP
  - Use actual database (test database, isolated from dev)
  - Test authentication, authorization, validation
  - Test response formats match API contracts
  - Test error scenarios (400, 401, 403, 404, 500)
  - Slower execution but critical for integration validation

**NestJS Testing Pattern:**
```typescript
// Unit Test Example (Service)
describe('AuthService', () => {
  let service: AuthService;
  let mockPrisma: DeepMockProxy<PrismaService>;
  let mockI18n: jest.Mocked<I18nService>;
  let mockLogger: jest.Mocked<Logger>;

  beforeEach(async () => {
    const module: TestingModule = await Test.createTestingModule({
      providers: [
        AuthService,
        { provide: PrismaService, useValue: mockDeep<PrismaService>() },
        { provide: I18nService, useValue: { t: jest.fn() } },
        { provide: Logger, useValue: { log: jest.fn(), error: jest.fn() } },
      ],
    }).compile();

    service = module.get<AuthService>(AuthService);
    mockPrisma = module.get(PrismaService);
    mockI18n = module.get(I18nService);
    mockLogger = module.get(Logger);
  });

  it('should throw error when user not found', async () => {
    mockPrisma.user.findUnique.mockResolvedValue(null);
    mockI18n.t.mockReturnValue('User not found');

    await expect(service.login({ email: 'test@test.com' }))
      .rejects
      .toThrow('User not found');

    expect(mockI18n.t).toHaveBeenCalledWith('errors.USER_NOT_FOUND');
  });
});
```

**E2E Test Pattern:**
```typescript
// E2E Test Example
describe('AuthController (e2e)', () => {
  let app: INestApplication;

  beforeAll(async () => {
    const moduleFixture: TestingModule = await Test.createTestingModule({
      imports: [AppModule],
    }).compile();

    app = moduleFixture.createNestApplication();
    await app.init();

    // Run migrations, seed test data
    await prisma.$executeRaw`TRUNCATE TABLE users CASCADE`;
  });

  afterAll(async () => {
    await app.close();
  });

  it('/auth/login (POST) - success', () => {
    return request(app.getHttpServer())
      .post('/auth/login')
      .send({ email: 'test@test.com', password: 'password' })
      .expect(200)
      .expect((res) => {
        expect(res.body).toHaveProperty('success', true);
        expect(res.body).toHaveProperty('status', 200);
        expect(res.body).toHaveProperty('data');
        expect(res.body.data).toHaveProperty('token');
        expect(res.body).toHaveProperty('message');
      });
  });
});
```

**Test Database Strategy:**
- Use separate test database (DATABASE_URL_TEST environment variable)
- Run migrations before e2e tests (beforeAll hook)
- Clean data between tests (afterEach or beforeEach truncate)
- Option 1: PostgreSQL test database (matching production)
- Option 2: SQLite in-memory (faster, but different from production)
- Recommendation: PostgreSQL test database for Epic 8.5 (consistency)

### Testing Standards Summary

**Test Organization:**
- **Unit tests**: Co-located with source files (`src/modules/*/___tests__/`)
- **E2E tests**: In `test/` directory at project root
- **Test utilities**: `test/utils/` for shared mocks and helpers

**Test Naming Conventions:**
- Unit test files: `*.service.spec.ts`, `*.controller.spec.ts`, `*.guard.spec.ts`
- E2E test files: `*.e2e-spec.ts`
- Describe blocks: Class or feature name (e.g., `'AuthService'`, `'Login endpoint'`)
- Test cases: Should be descriptive and start with 'should' (e.g., `'should throw error when user not found'`)

**Test Structure (AAA Pattern):**
```typescript
it('should create user successfully', async () => {
  // Arrange
  const createUserDto = { email: 'test@test.com', password: 'password' };
  mockPrisma.user.create.mockResolvedValue(mockUser);

  // Act
  const result = await service.createUser(createUserDto);

  // Assert
  expect(result).toEqual(mockUser);
  expect(mockPrisma.user.create).toHaveBeenCalledWith({
    data: createUserDto,
  });
});
```

**Coverage Requirements:**
- **Service Layer**: Minimum 80% coverage
- **Controller Layer**: Minimum 70% coverage
- **Critical Paths**: 100% coverage (authentication, authorization, payment, etc.)
- **Edge Cases**: Test error scenarios, boundary conditions, null/undefined handling

**Test Isolation Principles:**
- Each test is independent (no shared state)
- Use beforeEach to reset mocks and set up fresh state
- Use afterEach to clean up (database, files, etc.)
- No test should depend on another test's execution
- Tests should pass regardless of execution order

**Mock Strategy:**
- **External Services**: Always mock (Prisma, S3, email, SMS)
- **Internal Services**: Mock in unit tests, real in e2e tests
- **I18nService**: Mock in unit tests (return key), real in e2e tests
- **Logger**: Mock in unit tests (verify calls), real in e2e tests
- **Database**: Mock Prisma in unit tests, real database in e2e tests

**E2E Response Validation Pattern:**
```typescript
// Success Response
expect(response.body).toMatchObject({
  success: true,
  status: 200,
  data: expect.any(Object),
  message: expect.any(String),
});

// Paginated Response
expect(response.body).toMatchObject({
  success: true,
  status: 200,
  data: expect.any(Array),
  count: expect.any(Number),
  message: expect.any(String),
});

// Error Response
expect(response.body).toMatchObject({
  success: false,
  status: expect.any(Number),
  message: expect.any(String),
});
```
