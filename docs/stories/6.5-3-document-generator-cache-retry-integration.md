# Story 6.5.3: Document Generator Cache & Retry Integration

Status: done

## Story

As a developer,
I want DocumentGeneratorService'te cache ve retry mekanizmalarını entegre etmek,
so that PDF/Excel generation işlemleri cache'lensin ve geçici hatalar otomatik retry edilsin.

## Acceptance Criteria

1. **AC-6.5.3.1:** DocumentGeneratorService Cache Integration
   - `src/modules/document-generator/services/document-generator.service.ts` güncellendi
   - Constructor'da CacheService ve RetryService inject edilmiş
   - CacheService (Story 6.5-1'den) import edilmiş: `src/common/services/cache.service.ts`
   - RetryService (Story 6.5-2'den) import edilmiş: `src/common/services/retry.service.ts`

2. **AC-6.5.3.2:** Cache Key Generation implement edilmiş
   - Cache key pattern: `document:{format}:{templateName}:{dataHash}`
   - Example: `document:PDF:invoice:a1b2c3d4...` (SHA-256 hash)
   - CacheService.generateCacheKey() kullanılır:
     ```typescript
     const cacheKey = this.cacheService.generateCacheKey(
       'document',
       format, // 'PDF' | 'EXCEL'
       templateName,
       data
     );
     ```
   - Data hash otomatik olarak CacheService tarafından oluşturulur

3. **AC-6.5.3.3:** Cache Check before Generation
   - Document generation öncesi cache check
   - Cache hit durumunda: Cached result return edilir (generation skip edilir)
   - Cache miss durumunda: Normal generation workflow devam eder
   - Cache check example:
     ```typescript
     const cached = await this.cacheService.get<GenerationResult>(cacheKey);
     if (cached) {
       this.logger.log(`Cache hit for key: ${cacheKey}`);
       return cached;
     }
     ```

4. **AC-6.5.3.4:** Cache Write after Generation
   - Successful generation sonrası result cache'e yazılır
   - TTL: 1 hour (3600000ms) default, DOCUMENT_CACHE_TTL environment variable ile configurable
   - Cached value type: `GenerationResult` (Base64 string veya Buffer)
   - Cache write example:
     ```typescript
     await this.cacheService.set(cacheKey, result, cacheTTL);
     this.logger.log(`Cached result for key: ${cacheKey}`);
     ```

5. **AC-6.5.3.5:** Cache Write Failure Handling
   - Cache write failure: Log warning ama generation başarısız sayılmaz (non-blocking)
   - Try-catch wrapper ile cache write protected
   - Example:
     ```typescript
     try {
       await this.cacheService.set(cacheKey, result, cacheTTL);
     } catch (error) {
       this.logger.warn(`Cache write failed: ${error.message}`, { cacheKey });
       // Continue, don't fail generation
     }
     ```

6. **AC-6.5.3.6:** PDF Generation Retry Integration
   - Puppeteer PDF generation RetryService ile wrap edilmiş
   - RetryService.executeWithRetry() kullanılır
   - Retry configuration:
     - maxAttempts: 3 (DOCUMENT_RETRY_MAX_ATTEMPTS environment variable ile configurable)
     - baseDelay: 1000ms (DOCUMENT_RETRY_BASE_DELAY environment variable ile configurable)
     - context: `PDF Generation: ${templateName}`
   - Retry example:
     ```typescript
     const result = await this.retryService.executeWithRetry(
       async () => await this.generatePDFWithPuppeteer(html),
       {
         maxAttempts: this.retryMaxAttempts,
         baseDelay: this.retryBaseDelay,
         context: `PDF Generation: ${templateName}`
       }
     );
     ```

7. **AC-6.5.3.7:** Excel Generation Retry Integration
   - Excel generation RetryService ile wrap edilmiş
   - RetryService.executeWithRetry() kullanılır
   - Retry configuration: PDF ile aynı (maxAttempts: 3, baseDelay: 1000ms)
   - Retry example:
     ```typescript
     const result = await this.retryService.executeWithRetry(
       async () => await this.excelAdapter.generate(data),
       {
         maxAttempts: this.retryMaxAttempts,
         baseDelay: this.retryBaseDelay,
         context: `Excel Generation: ${templateName}`
       }
     );
     ```

8. **AC-6.5.3.8:** Error Handling
   - Cache miss: Generate document normally (null check)
   - Generation failure: Log error, throw DocumentGenerationException (mevcut error handling korunur)
   - Retry exhausted: Throw error with clear message (RetryService son error'ı throw eder)
   - Cache write failure: Log warning but continue (non-blocking)

9. **AC-6.5.3.9:** Logging
   - Cache hit: `[DocumentGenerator] Cache hit for key: document:PDF:invoice:...`
   - Cache miss: `[DocumentGenerator] Cache miss, generating document: PDF - invoice`
   - Generation start: `[DocumentGenerator] Generating PDF: invoice`
   - Retry attempts: `[DocumentGenerator] Retry attempt 2/3 for PDF: invoice` (RetryService logging)
   - Cache write: `[DocumentGenerator] Cached result for key: document:PDF:invoice:...`
   - Cache write failure: `[DocumentGenerator] Cache write failed: {error}` (warning level)

10. **AC-6.5.3.10:** Environment Configuration
    - `.env.example` dosyasına yeni environment variables eklendi:
      ```
      # Document Generator Cache & Retry Configuration
      DOCUMENT_CACHE_TTL=3600000          # 1 hour (ms)
      DOCUMENT_RETRY_MAX_ATTEMPTS=3       # Max retry attempts
      DOCUMENT_RETRY_BASE_DELAY=1000      # Base delay (ms)
      ```
    - ConfigService integration:
      - `DOCUMENT_CACHE_TTL`: Number, default 3600000 (1 hour)
      - `DOCUMENT_RETRY_MAX_ATTEMPTS`: Number, default 3
      - `DOCUMENT_RETRY_BASE_DELAY`: Number, default 1000

11. **AC-6.5.3.11:** Unit Tests
    - Test file: `src/modules/document-generator/services/__tests__/document-generator.service.spec.ts` güncellendi
    - Test scenarios:
      - Cache hit scenario: Cached result returned, generation skipped
      - Cache miss scenario: Generate document, cache result
      - Retry on failure: PDF generation retry on Puppeteer timeout
      - Retry on failure: Excel generation retry on adapter error
      - Retry exhaustion: Throw error after max attempts
      - Cache write failure: Continue without caching (non-blocking)
    - Mock CacheService and RetryService

12. **AC-6.5.3.12:** Integration Tests
    - Test file: `test/document-generator.integration-spec.ts` güncellendi (veya yeni oluşturuldu)
    - Test scenarios:
      - End-to-end PDF generation with cache miss
      - End-to-end PDF generation with cache hit
      - End-to-end Excel generation with cache miss
      - End-to-end Excel generation with cache hit
      - Retry integration with real RetryService

## Dev Notes

### Architecture Patterns and Constraints

**Service Integration Pattern:**
- DocumentGeneratorService, CacheService ve RetryService'i dependency injection ile alır
- CacheService: Generic cache operations (Story 6.5-1'den)
- RetryService: Exponential backoff retry (Story 6.5-2'den)
- [Source: docs/tech-spec-epic-6.5.md#Integration-Points]

**Cache-Aside Pattern:**
- Cache check before generation (cache hit → return cached, cache miss → generate)
- Cache write after successful generation
- Cache write failure: Non-blocking (application continues)
- [Source: docs/tech-spec-epic-6.5.md#Workflows-and-Sequencing]

**Retry Pattern:**
- Wrap external/expensive operations (Puppeteer, Excel generation) with retry
- Exponential backoff: 2^(attempt-1) * baseDelay
- Max attempts: 3 (configurable)
- Context-based logging for debugging
- [Source: docs/tech-spec-epic-6.5.md#Retry-Workflow]

**Error Handling Pattern:**
- Cache miss: Normal flow continues (null check)
- Cache write failure: Log warning, don't fail generation (non-blocking)
- Generation failure: Existing error handling korunur
- Retry exhausted: Throw last error with clear message
- [Source: docs/tech-spec-epic-6.5.md#Error-Handling]

**Configuration Management:**
- Environment variables: DOCUMENT_CACHE_TTL, DOCUMENT_RETRY_MAX_ATTEMPTS, DOCUMENT_RETRY_BASE_DELAY
- ConfigService ile configuration management
- Default values provided (1 hour cache TTL, 3 retry attempts, 1s base delay)
- [Source: docs/tech-spec-epic-6.5.md#Configuration-Management]

**Module Structure:**
- DocumentGeneratorService: `src/modules/document-generator/services/document-generator.service.ts`
- CacheService: `src/common/services/cache.service.ts` (Story 6.5-1)
- RetryService: `src/common/services/retry.service.ts` (Story 6.5-2)
- [Source: docs/architecture/project-structure.md#modules]

**Import Organization:**
- 8-group import order: Libraries → DTOs → Services → Repositories → Entities → Interfaces → Enums → Events
- [Source: docs/architecture/implementation-patterns-ai-agent-consistency-rules.md#Import-Organization]

### Learnings from Previous Stories

**From Story 6.5-1-common-cache-service-sha-256-hash-based (Status: done)**

- **CacheService Usage Pattern:**
  - CacheService.generateCacheKey() ile cache key oluştur
  - CacheService.get<T>() ile cache check
  - CacheService.set<T>() ile cache write
  - Generic type support: `<GenerationResult>`
  - [Source: stories/6.5-1-common-cache-service-sha-256-hash-based.md#CacheService-Methods]

- **Cache Key Generation:**
  - Pattern: `{prefix}:{hash(parts)}`
  - Example: `document:PDF:invoice:a1b2c3d4...`
  - SHA-256 hash otomatik oluşturulur (sortObjectKeys)
  - Consistent hash: Same data → same hash
  - [Source: stories/6.5-1-common-cache-service-sha-256-hash-based.md#Cache-Key-Format]

- **Cache Error Handling:**
  - Cache miss: null return (no exception)
  - Cache failures: Gracefully handled (fallback to no-cache)
  - Cache write failure: Log warning, continue (non-blocking)
  - [Source: stories/6.5-1-common-cache-service-sha-256-hash-based.md#Error-Handling]

**From Story 6.5-2-common-retry-service-exponential-backoff (Status: done)**

- **RetryService Usage Pattern:**
  - RetryService.executeWithRetry() ile operation wrap et
  - Retry options: maxAttempts, baseDelay, context
  - Context string for logging: `PDF Generation: ${templateName}`
  - [Source: stories/6.5-2-common-retry-service-exponential-backoff.md#RetryService-Methods]

- **Exponential Backoff:**
  - Formula: `2^(attempt-1) * baseDelay`
  - Delay sequence: 0ms → 1000ms → 2000ms → 4000ms...
  - First attempt: Immediate (no delay)
  - [Source: stories/6.5-2-common-retry-service-exponential-backoff.md#Exponential-Backoff-Pattern]

- **Retry Logging:**
  - Each attempt: `[context] Attempt 1/3`
  - Retry delay: `[context] Waiting 1000ms before retry attempt 2`
  - Success on retry: `[context] Succeeded on attempt 2/3`
  - Final failure: `[context] All 3 attempts failed`
  - [Source: stories/6.5-2-common-retry-service-exponential-backoff.md#Logging-Integration]

**From Story 6-4-document-generation-endpoints (Epic 6)**

- **DocumentGeneratorService Structure:**
  - Already has ConfigService, LoggerService
  - generate() method returns GenerationResult
  - Puppeteer for PDF, ExcelAdapter for Excel
  - [Source: docs/epics/epic-6-document-generation-adapter-pattern-hrsync-backend.md#Story-6.4]

- **Document Generation Flow:**
  - Template selection → Data validation → Generation → S3 upload → Return result
  - GenerationResult: { url: string, format: string, size: number }
  - [Source: docs/tech-spec-epic-6.5.md#Integration-Sequence]

**Key Takeaways:**
- Story 6.5-3 integrates Story 6.5-1 (CacheService) and Story 6.5-2 (RetryService) into DocumentGeneratorService
- Cache-Aside pattern: Check cache before generation, write after successful generation
- Retry pattern: Wrap Puppeteer and Excel generation with retry mechanism
- Non-blocking cache writes: Cache write failures don't fail generation
- Environment-based configuration: Cache TTL, retry attempts, base delay
