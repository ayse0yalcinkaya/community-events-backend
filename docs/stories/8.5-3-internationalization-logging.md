# Story 8.5.3: Internationalization & Logging

Status: done

## Story

As a developer,
I want i18n ve logger kullanımı tüm servis ve controller'larda tam olarak entegre edilmiş olsun,
so that multi-language support tam olsun ve console.log kullanılmasın, profesyonel logging standardı uygulanmış olsun.

## Acceptance Criteria

1. **AC-8.5-3.1:** Tüm Services i18n kullanıyor
   - Constructor'da `I18nService` inject edilmiş
   - Hata mesajları i18n ile translate ediliyor:
     ```typescript
     throw new BadRequestException(
       this.i18n.t('errors.USER_NOT_FOUND')
     );
     ```
   - Success mesajları i18n ile translate ediliyor
   - Tüm service'ler güncellenmiş: AuthService, UsersService, PermissionsService, FilesService, TokenService

2. **AC-8.5-3.2:** Tüm Controllers i18n kullanıyor
   - Response message'lar i18n ile translate ediliyor
   - Validation error mesajları translate ediliyor
   - Tüm controller'ler güncellenmiş: AuthController, UsersController, ProfileController, PermissionsController, FilesController

3. **AC-8.5-3.3:** i18n translation dosyaları güncellendi
   - `src/modules/i18n/translations/tr/` - Türkçe translation'lar
   - `src/modules/i18n/translations/en/` - İngilizce translation'lar
   - Tüm hata kodları ve mesajlar tanımlı:
     - `errors.json` - Hata mesajları (USER_NOT_FOUND, INVALID_CREDENTIALS, etc.)
     - `success.json` - Başarı mesajları (OPERATION_SUCCESSFUL, CREATED, UPDATED, DELETED)
     - `validation.json` - Validasyon mesajları

4. **AC-8.5-3.4:** Console.log'lar Logger service'e dönüştürüldü
   - `console.log` → `this.logger.log`
   - `console.error` → `this.logger.error`
   - `console.warn` → `this.logger.warn`
   - `console.debug` → `this.logger.debug`
   - Hiçbir `console.*` kullanımı kalmadı (main.ts bootstrap hariç)

5. **AC-8.5-3.5:** Logger Service tüm dosyalarda kullanılıyor
   - Services
   - Controllers
   - Guards
   - Interceptors
   - Middleware (eğer varsa)
   - Exception Filters

6. **AC-8.5-3.6:** Logger context eklendi
   ```typescript
   constructor(
     private readonly logger: Logger = new Logger(ServiceName.name)
   ) {}
   ```
   - Her service ve controller'da doğru context ismi kullanılıyor
   - Logger constructor'da initialize ediliyor

7. **AC-8.5-3.7:** Translation key organization standardı uygulandı
   - Hata mesajları: `errors.SPECIFIC_ERROR_CODE` formatında
   - Başarı mesajları: `success.OPERATION_TYPE` formatında
   - Validasyon mesajları: `validation.FIELD_RULE` formatında
   - Key isimlendirme consistent ve anlamlı

8. **AC-8.5-3.8:** AcceptLanguage header'dan dil algılanıyor
   - i18n module doğru configure edilmiş
   - Resolver'lar doğru sırada (header, cookie, query)
   - Default language fallback çalışıyor
   - `Accept-Language: tr` → Türkçe response
   - `Accept-Language: en` → İngilizce response

9. **AC-8.5-3.9:** Logging standardı uygulandı
   - Info log'lar: Method entry/exit points
   - Debug log'lar: Detaylı debugging bilgisi
   - Error log'lar: Exception details ve stack traces
   - Warn log'lar: Potential issues
   - Log message'lar descriptive ve context içeriyor

10. **AC-8.5-3.10:** Build ve testler başarılı
    - `npm run build` hata vermeden çalışıyor
    - `npm run type-check` başarılı
    - Tüm TypeScript compile error'ları çözüldü
    - Application başlatılıyor ve çalışıyor
    - Hiçbir runtime error yok

## Dev Notes

### Architecture Patterns and Constraints

**i18n Integration Pattern:**
- i18n Epic 7.1'de kurulmuş, bu story'de tam entegrasyon yapılacak
- `I18nService` constructor'da inject edilir
- Translation key formatı: `category.SPECIFIC_KEY` (örn: `errors.USER_NOT_FOUND`)
- `this.i18n.t('key')` kullanılarak translate edilir
- AcceptLanguage header'dan dil algılanır (HeaderResolver, AcceptLanguageResolver)
- Fallback language: 'en' (İngilizce)
- [Source: Epic 7.1 implementation, docs/epics/epic-7-developer-infrastructure.md]

**Logger Integration Pattern:**
- Logger Epic 7.3'te kurulmuş (Winston logger)
- `Logger` instance constructor'da initialize edilir: `new Logger(ContextName.name)`
- Her class kendi context'ini kullanır (AuthService, UsersController, vb.)
- Log levels: `log()` (info), `debug()`, `error()`, `warn()`
- Error logging stack trace içermeli
- Console.log kullanımı YASAK (sadece main.ts bootstrap'ta allowed)
- [Source: Epic 7.3 implementation, docs/epics/epic-7-developer-infrastructure.md]

**Translation File Structure:**
```
src/modules/i18n/translations/
├── tr/
│   ├── errors.json
│   ├── success.json
│   └── validation.json
└── en/
    ├── errors.json
    ├── success.json
    └── validation.json
```

**Service i18n Integration Example:**
```typescript
import { Injectable, BadRequestException } from '@nestjs/common';
import { I18nService } from 'nestjs-i18n';
import { Logger } from '@nestjs/common';

@Injectable()
export class AuthService {
  private readonly logger = new Logger(AuthService.name);

  constructor(
    private readonly i18n: I18nService,
    // ... other dependencies
  ) {}

  async login(credentials: LoginDto) {
    this.logger.log('Login attempt for user: ${credentials.email}');

    const user = await this.findUser(credentials.email);
    if (!user) {
      this.logger.warn(`Login failed: User not found - ${credentials.email}`);
      throw new BadRequestException(
        this.i18n.t('errors.USER_NOT_FOUND')
      );
    }

    // ... rest of logic
    this.logger.log(`Login successful for user: ${user.id}`);
    return { token, user };
  }
}
```

**Controller i18n Integration Example:**
```typescript
import { Controller, Post, Body } from '@nestjs/common';
import { I18nService } from 'nestjs-i18n';
import { Logger } from '@nestjs/common';

@Controller('auth')
export class AuthController {
  private readonly logger = new Logger(AuthController.name);

  constructor(
    private readonly authService: AuthService,
    private readonly i18n: I18nService,
  ) {}

  @Post('login')
  async login(@Body() credentials: LoginDto) {
    this.logger.log(`Login request received for: ${credentials.email}`);

    const result = await this.authService.login(credentials);

    return {
      data: result,
      message: this.i18n.t('success.LOGIN_SUCCESSFUL')
    };
  }
}
```

**Exception Filter i18n Integration:**
```typescript
import { ExceptionFilter, Catch, ArgumentsHost } from '@nestjs/common';
import { I18nService } from 'nestjs-i18n';
import { Logger } from '@nestjs/common';

@Catch()
export class AllExceptionsFilter implements ExceptionFilter {
  private readonly logger = new Logger(AllExceptionsFilter.name);

  constructor(private readonly i18n: I18nService) {}

  catch(exception: any, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse();
    const status = exception.getStatus?.() || 500;

    const message = exception.message || this.i18n.t('errors.INTERNAL_SERVER_ERROR');

    this.logger.error(
      `Exception caught: ${exception.message}`,
      exception.stack
    );

    response.status(status).json({
      success: false,
      status,
      message: this.i18n.t(message), // Translate if it's a key
      errors: exception.errors || undefined
    });
  }
}
```
