# Story 6.5.1: Common Cache Service (SHA-256 Hash-Based)

Status: done

## Story

As a developer,
I want proje genelinde kullanılabilir hash-based caching system,
so that tüm modüllerde cache ihtiyacı olan operasyonlar için reusable cache servisi kullanabilirim.

## Acceptance Criteria

1. **AC-6.5.1:** CacheService oluşturulmuş
   - `src/common/services/cache.service.ts` dosyası oluşturulmuş
   - Dependencies: NestJS `@nestjs/cache-manager`, Cache instance (injected via CACHE_MANAGER)
   - Default TTL: 1 hour (3600000ms), configurable per operation
   - Generic type support: `<T>` for any cached value type

2. **AC-6.5.2:** generateCacheKey() Method implement edilmiş
   - Method signature: `generateCacheKey(prefix: string, ...parts: any[]): string`
   - Generic key generation: `{prefix}:{hash(parts)}`
   - Data hash: SHA-256 of sorted JSON (sortObjectKeys)
   - Example outputs: `doc:PDF:invoice:a1b2c3d4...`, `user:profile:123:abc...`
   - Consistent hash: `{a:1, b:2}` and `{b:2, a:1}` produce same hash

3. **AC-6.5.3:** get<T>() Method implement edilmiş
   - Method signature: `get<T>(key: string): Promise<T | null>`
   - Retrieve cached value with generic type support
   - Cache miss durumunda null return eder (error throw etmez)
   - Generic type support: Works with any TypeScript type

4. **AC-6.5.4:** set<T>() Method implement edilmiş
   - Method signature: `set<T>(key: string, value: T, ttl?: number): Promise<void>`
   - Store value with optional TTL
   - Default TTL: 3600000ms (1 hour) if not provided
   - Generic type support: Works with any TypeScript type

5. **AC-6.5.5:** delete() Method implement edilmiş
   - Method signature: `delete(key: string): Promise<void>`
   - Invalidate cache entry
   - Cache miss durumunda error throw etmez (idempotent)

6. **AC-6.5.6:** clear() Method implement edilmiş
   - Method signature: `clear(): Promise<void>`
   - Clear all cache entries
   - Redis store'daki tüm cache entries'i temizler

7. **AC-6.5.7:** SHA-256 Hash Generation implement edilmiş
   - `generateDataHash(data: any): string` (private method)
   - Sort object keys recursively (consistent hash için)
   - JSON.stringify sorted data
   - SHA-256 hash (64 hex characters)
   - Node.js `crypto.createHash('sha256')` kullanılır
   - `sortObjectKeys(obj: any): any` (private method)
     - Recursive key sorting
     - Handles arrays, objects, primitives
     - Ensures consistent hash for same data regardless of key order

8. **AC-6.5.8:** NestJS Cache Module Integration implement edilmiş
   - Global CacheModule.register() in AppModule veya CommonModule
   - Redis store configuration:
     - Store: Redis (cache-manager-redis-store)
     - Redis client: ioredis
     - Configuration: Environment variables (REDIS_HOST, REDIS_PORT, REDIS_PASSWORD, REDIS_DB)
     - TTL: 3600 seconds (1 hour) - default
     - Connection pooling: Redis connection pool management
   - Cache failures gracefully handled (fallback to no-cache, error handling)

9. **AC-6.5.9:** CommonModule Export implement edilmiş
   - CacheService exported from `src/common/services/index.ts`
   - Available to all modules via CommonModule import
   - CommonModule üzerinden tüm modüllere export edilir

10. **AC-6.5.10:** Unit Tests implement edilmiş
    - Test file: `src/common/services/__tests__/cache.service.spec.ts`
    - Test coverage: > 80%
    - Test scenarios:
      - Key generation with various input types
      - Cache get/set/delete/clear operations
      - TTL expiration behavior
      - Hash consistency (same data → same hash)
      - Generic type support
      - Cache miss scenarios (null return)
      - Cache failures gracefully handled

11. **AC-6.5.11:** Integration Tests implement edilmiş
    - CacheModule registration test
    - CommonModule export/import test
    - Redis mock/container integration test
    - Cache hit/miss in DocumentGeneratorService integration test

## Dev Notes

### Architecture Patterns and Constraints

**Service Pattern:**
- CacheService, generic cache operations sağlar
- SHA-256 hash-based key generation ile consistent cache keys
- Generic type support ile herhangi bir veri tipini cache'leyebilir
- TTL support ile cache expiry management
- [Source: docs/tech-spec-epic-6.5.md#Services-and-Modules]

**Dependency Injection Pattern:**
- NestJS dependency injection kullanılır
- Cache instance CACHE_MANAGER token ile inject edilir
- LoggerService optional dependency (Epic 7.3'ten)
- [Source: docs/tech-spec-epic-6.5.md#System-Architecture-Alignment]

**Hash-Based Key Generation:**
- SHA-256 hash ile consistent cache keys
- Recursive key sorting ile `{a:1, b:2}` and `{b:2, a:1}` aynı hash üretir
- Prefix-based key namespace: `doc:PDF:invoice:hash`
- [Source: docs/tech-spec-epic-6.5.md#Data-Models-and-Contracts]

**Error Handling Pattern:**
- Cache miss durumunda null return eder (error throw etmez)
- Cache failures gracefully handled (fallback to no-cache)
- Redis connection failures gracefully handled
- [Source: docs/tech-spec-epic-6.5.md#Reliability/Availability]

**Module Structure:**
- CacheService: `src/common/services/cache.service.ts`
- CommonModule export: `src/common/services/index.ts`
- [Source: docs/architecture/project-structure.md#common]

**Import Organization:**
- 8-group import order: Libraries → DTOs → Services → Repositories → Entities → Interfaces → Enums → Events
- [Source: docs/architecture/implementation-patterns-ai-agent-consistency-rules.md#Import-Organization]
